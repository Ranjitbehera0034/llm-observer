# GitHub Issue Labels

Use the following labels to categorize issues and PRs:

- `bug`: Something isn't working.
- `feature`: New functionality.
- `security`: Security-related fixes or improvements.
- `sprint-1`: Foundation & Credibility.
- `sprint-2`: Developer Experience.
- `sprint-3`: Enterprise Readiness.
- `sprint-4`: Observability Layer.
- `sprint-5`: Ecosystem & Reach.
- `sprint-6`: Monetization & v2.0.
- `good-first-issue`: Simple tasks for new contributors.
